Getting started with Replit

In order to actually submit your code challenge, you'll need to use Replit. Which might lead you to wonder, "What is that?" Replit is a site that lets you write and "run" (that means activate) your code in a browser. Laptops, tablets, and phones will all work, though we recommend a laptop. Check out the video below about how to get started with the site.

How The Lessons Work

Each lesson is going to feature a primary article and video. They will largely cover the same topics, the videos are just to help those who prefer to learn from videos instead of blogs. Real quick though, I want to introduce some article features.

Code Screens and Outputs

Whenever we write code, it'll appear in a mini screen like this. 

let greeting = "Hello there!"
console.log(greeting)

And the output of that code (if there is any) may appear in an output window like this. 

Hello there!

Sometimes though, if the output is very tiny, it might just be put as a note next to the original code.

CodePens

Those windows are nice, but you can't activate the code. Enter CodePens!

To run this codepen, make sure you're connected to the internet Untitled

These are snippets of code that you can read, edit, and run. Since they need an internet connection to use, any code written in them will also be written above in a code screen as well. To edit CodePens, simply tap the big "Run Pen" button in the "Result" window. If you are on a phone, they'll collapse into a single screen, so you'll need to click "JS" to edit the code, and "Result" to see outputs.

The code should automatically rerun when you edit it, but you can also click the "Rerun" button if it doesn't update. Though, if you have an error it may not tell you, so be sure to double check your code for typos if it stops running!

CodePens Don't Save!

Your CodePen work only exists in your browser. If you ever edit something cool, be sure to copy it out into a Replit or some other file! Otherwise it will be lost when you leave the page!

Jargon Definitions

Most words are defined in context in their lessons, but here are the definitions of some more "programmy" words you might be unfamiliar with.

Syntax

This isn't quite the same thing as the grammar version that you know. In programming, syntax refers to the exact way a piece of code is structured. It's important because every language does have the same ideas, but they are expressed with different words and symbols. Here's a JavaScript example: 

console.log('hi!')

That exact same code in C++ has this syntax instead: 

#include <iostream>
int main() {
    std::cout << "Hello World!";
    return 0;
}

As you can see, JavaScript Syntax is a bit better for beginners.

The Console

We need somewhere to see our programs' output. Since we're only making text based programs, the output won't be something like a website. Instead it will be the "Console." As far as we're concerned the console is just a screen to print things onto. Replit and CodePen both have screens to show you exactly what you print.

Debug

Most of programming is actually just fixing broken programs. When we are trying to figure out why something isn't working, that's called "debugging." You will spend a vast majority of your time debugging if you become a developer.
