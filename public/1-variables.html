Lesson 1: Variables

How To Save Information

In programming, we need ways to save and reuse pieces of info. One way we can store data is by using a variable. Here is an example of a variable called myMood that has a value of Happy:

See how we set the value of myMood by using the = operator. But what's that let doing out in the front? In JavaScript, in order to declare, or "create", a variable you have to use a special keyword first. The two modern options are let or const.

let allows change

If it's ok for the variable to change its value, for example 100 to 101, then we use the let keyword. 

let myNumber = 100
myNumber = 101

Notice that when we changed the variable's value, we didn't use another let keyword. We only use the let keyword once.

const means constant

If we want to lock the value in and never change it, we would use the const keyword. 

const myNumber = 100
myNumber = 101
THIS WILL THROW AN ERROR

So which to use?

I recommend you stick to let when you're starting out. const reassigning errors aren't hard to spot, but "simple" problems are still annoying. Once you're more comfortable, you can start using const properly, but let is ok for variables now.

No using before declaring!

Regardless of let or const, don't forget that you can't reference a variable before you've actually created it! 

myNumber = 100
let myNumber = 101
NO GOOD, YOU USED myNumber BEFORE YOU DEFINED IT

Variable Name Rules

Here are some starter rules for naming your variables

  • Names can only include letters, numbers, and underscores
  • You can't start names with numbers
  • No spaces allowed

Data Types

So far we've only used numbers or text for our variables, but those aren't the only things we can store. There are many different kinds of data types in JavaScript, but here are some of the simple ones to start us off.

Numbers

JavaScript only has one main Number type. Both whole numbers (called "Integers") and decimals (called "Float") are just Numbers. 

12
14242.23

You can also do math with JavaScript. Here are some of the basic operations: 

+ = addition
- = subtraction
/ = division
* = multiplication

But unlike math, variables can reference themselves. When making a variable bigger, it can check its current value to calculate a new value. 

let a = 5
a = a + 10

Our variable a is now 15. I know the "referencing itself" thing is weird, but you get used to it. It's so common, there's a special shortcut operator. 

let a = 5
a += 10

Once again, a is now 15. Basically, The +=, -=, /=, and *= assignment operators are a great way to increment or decrement values.

Strings

In JavaScript a piece of text is called a String. It's a "string" because a piece of text is like a bunch of individual characters "strung" together. Strings are created by putting anything between double (") or single quotes('). Either is fine, just make sure they match! 

let string = "I am a string"
let otherString = 'So Am I'

The only tricky thing with strings at this stage is dealing with words like "don't". Be sure to use double quotes for the whole string so JavaScript doesn't end it at the apostrophe.

Booleans

That is a wonky name, but it's just what we call true and false. Also called a "bool" for short. In JavaScript, bools are lowercase. Don't put them in quotes, they aren't text, they're actual values. 

let myBoolean = true
myBoolean = false

Bools are for something called Flow Control, but more on that in Lesson 3!

Undefined and Null

Sometimes the value we want to save is ...no value. That's what undefined and null are. There is a difference between them, but it's irrelevant for now. 

let nothing = undefined
let alsoNothing = null

Undefined errors

Sometimes when JavaScript expects something but gets nothing, the value will show up as undefined. This is your hint that something is going wrong in your program and you should investigate further.

Comments

These aren't really a data type exactly, but they are useful. A comment is a chunk of text that is not seen by the program. It's useful for writing notes to yourself or your team members. 

// I am a single line comment

A Quick Peek Ahead

There are 2 slightly more complex data types called Objects and Arrays. An Array is a list of things. 

let people = ['Bob', 'Sara', 'Jake']

They're great, but we don't need them yet. The one we will kind of need is the Object. An Object is a bunch of key and value pairs. Where a key (usually called a property) points to a specific value.

Or to put it another way, use dot notation with a property name to get a value. 

let bob = {
  name: 'Bob',
  age: 32,
  isCool: true,
}
bob.name
// This would be 'Bob'
bob.age
// this would be 32
bob.isCool
// and of course this would be true

We will not be writing any objects ourselves just yet, we'll only read some built in objects that JS gives us. Remember, to access a value on an object, put a "." and then the property name.
