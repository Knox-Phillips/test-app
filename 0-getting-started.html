Getting started with Replit

In order to actually submit your code challenge, you'll need to use Repl.it. Which might lead you to wonder, "What is that?" Repl.it is a really intuitive site that lets you write code in a browser, and then run it. That means any internet connected device will let you edit code. Laptops, tablets, even phones will all work, though I recommend using a laptop or tablet because we'll need to use special characters. Code Screens and Outputs

Whenever we write code, it'll appear in a mini screen like this. 

let greeting = "Hello there!"
console.log(greeting)

And the output of that code (if there is any) may appear in an output window like this. 

Hello there!

Sometimes though, if the output is very tiny, it might just be put as a note next to the original code.

CodePens

Those windows are nice, but you can't activate the code. Enter CodePens!

To run this codepen, make sure you're connected to the internet Untitled

These are snippets of code that you can read, edit, and run. Since they need an internet connection to use, any code written in them will also be written above in a code screen. To edit them, simply tap the big "Run Pen" button in the "Result" window. If you are on a phone, they'll collapse into a single screen, so you'll need to click "JS" to edit the code, and "Result" to see outputs.

The code should automatically rerun when you edit it, but you can also click the "Rerun" button if it doesn't update. Though, if you have an error it may not tell you, so be sure to double check your code for typos if it stops running!

CodePens Don't Save!

Your codepen work only exists in your browser. If you ever edit something cool, be sure to copy it out into a Replit or some other file! Otherwise it will be lost when you leave the page!

Jargon Definitions

Feel free to skip this, as most words are defined in context in lessons, but here are the definitions of some more programmy-words that you might run into

Dynamic/Static/Hard Coded

Dynamic usually refers to information that can change. Like a function that greets people, the ability to change the names of the people it greets means its a dynamic function. Static means just the opposite: unable to change. And if a piece of information is static because we are explicitly writing it to be that way, you would say that the information is "hard coded" into the program.

Syntax

This isn't quite the same thing as grammar that your used to. In programming, syntax refers to the exact way a piece of code is written. It's important because every language does have the same ideas, but they are expressed with different words and symbols. 

console.log('hi!')

In C++ the syntax for the same idea would be 

#include 
int main() {
    std::cout << "Hello World!";
    return 0;
}

As you can see, JavaScript Syntax is a bit friendlier to beginners.

Iterations

We're going to learn about loops. In a programming loop, if you want to do something 3 times, each of those individual chunks are called iterations. So you'd say the first iteration, then second iteration, and the third iteration would be the last.

Mutable/Immutable

Mutable comes from the word Mutation, and it is a fancy pants way to say "something has the ability to change." If a piece of information is unable to change, it's said to be "immutable."
