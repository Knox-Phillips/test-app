Lesson 3: Flow Control

If This, Then What?

It's very common to have a program that changes its behavior based off of a set of different inputs. There are a bunch of ways to alter actions, but easily one of the most universal methods is the If/Else Statement. Check it out below, how do you think it works? 

const sayMood = (myMood) => {
  if (myMood === "Happy") {
    console.log("I'm feeling great!")
  } else if (myMood === "Sad") {
    console.log("I'm feeling pretty bummed")
  } else {
    console.log("I'm feeling...something")
  }
}

sayMood('Happy')

Don't worry, I'll explain it, but why not take a second just to explore and see what you can figure out on your own? If you're online, use Replit or the CodePen to play with the code below. How would you make the function log I'm feeling...something? What do you think that === thing is doing?

Structure of If/Else

Ok, done experimenting? Let's break it down then.

  • An If/Else Statement can be made up of 3 kinds of checks: if, else if, and else checks
  • Each type of check will only fire if a certain condition evaluates to true
  • You can use only one if and else check per statement, but you can use as many else if checks as you want
  • else checks are optional, since sometimes you don't want a catch-all situation. 
if (condition A) {
  // Runs if Condition A is true
} else if (condition B) {
  // Runs if Condition B is true
} else if (condition C) {
  // Runs if Condition C is true
} else {
  // Runs if ALL of the conditions are false
}

Conditions and Comparisons

Let's start simple with just an if statement to talk about conditions. By the way, it's perfectly valid (and common) to only use one if statement in your code. Anyway, a condition in JavaScript is any expression that evaluates to true or false. We do this by using the comparison operators:

OperatorNameExample
===Equal To'Hi' === 'Hi'
!==Not Equal To6 !== 7
>Greater Than10 > 1
<Less Than3 < 5
>=Greater Than Or Equal To5 >= 5
<=Less Than Or Equal To2 <= 2

The bottom 4 are mainly for numbers, but the first two can be used with any values really. So what you would do is take 2 values, put a comparison between them, and then stick that whole thing in the () of the if statement. If the condition is true, then the code in the {} will execute. But if the condition is false, then the code is ignored. Try it out below by playing around with the values. 

let name = 'bob'
let age = 32
if (name !== 'jo') {
  console.log('Hello, not Jo.')
}

Storing Conditionals

Conditionals evaluate to booleans, which are just values. That means that you can store conditions as variables. This is super common. So common in fact, that there's a Naming Convention devoted to storing booleans in variables and returning them in functions.

If your variable is a boolean, it should start with "is, has, can, could, should". Honestly it's usually just "is" or "can". If it's a function that returns a variable, you can just put "get" in front of that. And yes, that can sometimes sounds a little weird, but that's ok. 

let name = 'bob'
let age = 32
let canVote = age >= 18

console.log('canVote:', canVote)
if (canVote) {
  console.log("Let's go vote!")
}

const getCanVote = (personAge) => {
  return personAge >= 18
}

if (getCanVote(17)) {
  console.log("Let's go vote!")
}

Reversing Conditionals with !

One last thing on conditionals is that sometimes you want the opposite of one. If something is true, make it false. The way we do this is with the ! operator (prounounced bang). By putting it next to a value, it will invert it. So !true is false, and !false is true. 

let age = 32
let canVote = age >= 18
let cantVote = !canVote
console.log("can't vote:", cantVote)

// or directly on a conditional
let alsoCantVote = !(age >= 18)
console.log("alsoCantVote", alsoCantVote)

You may notice that we used () in the second example. That's because we have to be careful with what we want to reverse. if we did !age >= 18 then that would mean we only reversed the boolean value of age. Sometimes that's what we want, but when you want the entire conditional, wrap it up in a ().

Truthy and Falsy Values

Pop quiz: what do you think would happen if we did this: 

if ("hi there") {
  console.log('???')
}

We just put a non-boolean String into our conditional. Does this break? No, actually. That's because when you put a non-boolean value into the conditional slot JavaScript converts it into true if the value is truthy and false if the value is falsy. Yes, those are real terms in programming.

So, what's truthy and what's falsy? Here are all the falsy values in the language

Value Example
Literal Falsefalse
An Empty String""
The Number Zero0
The "Nothing" Valuesnull, undefined
Not a NumberNaN

That's it, everything else in JavaScript is a truthy value. All positive numbers are truthy, all negative numbers are truthy. All full strings are truthy, even strings made up of nothing but spaces like " ". EVERYTHING else is truthy. Play around with truthy and falsy values below.

else if and else

Now that we fully understand conditionals, let's get back to building our If/Else Statement. If our question boils down to "Do this or that", then we'll just need an else statement. 

let chanceOfRain = 72;
if (chanceOfRain >= 70) {
  console.log('Bring an umbrella.')
} else {
  console.log('No umbrella!')
}

An else is used when all other cases should result in that code block running. That's why there's no conditional next to the else. All you're saying is, "If the main conditional is false, run the else block." But this is binary, what if we have a second, third or many more options? That's where the else if conditions come in

An else if condition allows you to specify for more conditionals. 

let pet = 'cat'

if (pet === 'cat') {
  console.log("My favorite pet!")
} else if (pet === "dog") {
  console.log("My 2nd favorite pet!")
} else if (pet === "snake") {
  console.log("Hard pass.")
} else if (pet === "turtle") {
  console.log("Love 'em!")
} else {
  console.log("Your pet is meh.")
}

Remember, each If statement starts a new chain. And in each chain, only one code block can execute. So, it helps to be specific like this sometimes.

Reassigning variables

Look again at that above example. That's a lot of console.logs, right? What if we just saved a variable called msg (that's a universal shorthand for "message"), and then logged that only at the end? 

let pet = "cat"
let msg = ""

if (pet === "cat") {
  msg = "My favorite pet!"
} else if (pet === "dog") {
  msg = "My 2nd favorite pet!"
} else if (pet === "snake") {
  msg = "Hard pass."
} else if (pet === "turtle") {
  msg = "Love 'em!"
} else {
  msg = "Your pet is meh."
}
console.log(msg);

That's a little cleaner, but it looks like we added lines. Well we aren't done yet. See, if we use a variable, instead of using an empty value, we can set its initial value as the else value. Then we can remove else section. 

let pet = "cat"
let msg = "Your pet is meh."

if (pet === 'cat') {
  msg = "My favorite pet!"
} else if (pet === "dog") {
  msg = "My 2nd favorite pet!"
} else if (pet === "snake") {
  msg = "Hard pass."
} else if (pet === "turtle") {
  msg = "Love 'em!"
}
console.log(msg);

There, our code now does exactly the same thing, but it's easier on the eyes! When you change the way code is written, but still leave it's functionality alone, that's called refactoring.

Return vs else

There's a another case were we don't need an else statement and that's when we're returning in a function. As you know, the return keyword causes us to skip whatever was left in the function. So if you are returning conditionally, then you may not need an else. 

const petOpinion = (pet) => {
  if (pet === 'cat') {
    return "My favorite pet!"
  } else {
    return "Your pet is meh."
  }
}

// that's the same as:
const petOpinion = (pet) => {
  if (pet === 'cat') {
    return "My favorite pet!"
  }
  return "Your pet is meh."
}

We don't need the else statement in this case because if we hit the return in the "cat" conditional, we're getting out of the function anyway. It just keeps our function a little cleaner.

Multiple conditionals

So we have one conditional per block, yes, but what about a second conditional? Don't you worry, we have special operators to deal with combinations.

The OR operator ||

If you want something to happen as long as at least one condition is true, you should use the OR operator. It's just two "pipe" characters ||. On most keyboards you get that by holding shift and the "\" key, which is usually one above the enter key. Here's how you use it: 

let pet = "cat"
let msg = "Your pet is meh."

if (pet === 'cat' || pet === 'dog') {
  msg = "I love your pet!"
}
console.log(msg);

That doesn't seem terribly intuitive, I know that I wished it was like: 

if (pet === 'cat' || 'dog') {
  msg = "I love your pet!"
}
NO THAT'S WRONG

But what you have to remember is conditionals are each their own boolean, so the OR operator is saying either conditional, not either value.

The AND operator &&

If you want every single condition to be true, then you need the AND operator, which is just two ampersands. 

let pet = "dog"
let name = "fido"
let msg = "You are creative"
if (name === "fido" && pet === "dog") {
  msg = "Ya basic"
}
console.log(msg)

So, if your pet is a dog but it's name isn't "fido", then that if statement won't fire.
