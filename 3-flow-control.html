Lesson 3: Flow Control

If This, Then What?

Most programs change their behavior based off of a set of different inputs. There are many ways to alter actions, but one of the most universal is the If/Else Statement. 

const sayMood = (myMood) => {
  if (myMood === "Happy") {
    console.log("I'm feeling great!")
  } else if (myMood === "Sad") {
    console.log("I'm feeling pretty bummed")
  } else {
    console.log("I'm feeling...something")
  }
}

sayMood('Happy')

Structure of an If/Else

Ok, Let's break it down.

  • An If/Else Statement can be made up of 3 kinds of checks: if, else if, and else
  • Each type of check will only fire if a certain condition evaluates to true
  • You can use only one if and else checks per statement, but you can use as many else if checks as you want
  • else checks are optional, since sometimes you don't want a catch-all situation. 
if (condition A) {
  // Runs if Condition A is true
} else if (condition B) {
  // Runs if Condition B is true
} else if (condition C) {
  // Runs if Condition C is true
} else {
  // Runs if ALL of the conditions are false
}

Conditions and Comparisons

Let's start simple with just an if statement to talk about conditions. A condition in JavaScript is any expression that evaluates to true or false. We do this by using the comparison operators:

OperatorNameExample
===Equal To'Hi' === 'Hi'
!==Not Equal To6 !== 7
>Greater Than10 > 1
<Less Than3 < 5
>=Greater Than Or Equal To5 >= 5
<=Less Than Or Equal To2 <= 2

What you do is take 2 values, put an operator between them, and then stick that whole thing in the () of the if statement. If the condition is true, then the code in the next {} will execute. But if the condition is false, then the code doesn't run. 

let name = 'bob'
let age = 32
if (name !== 'jo') {
  console.log('Hello, not Jo.')
}

Storing Conditionals

Conditionals evaluate to booleans, which are just values. That means that you can store conditions as variables. If you do, there's a Naming Convention devoted to storing booleans in variables and returning them in functions.

The variable should start with "is, has, can, could, should". If it's a function that returns a bool, you can just put "get" in front of that. And yes, that looks weird, it's an optional convention. 

let name = 'bob'
let age = 32
let canVote = age >= 18

console.log('canVote:', canVote)
if (canVote) {
  console.log("Let's go vote!")
}

const getCanVote = (personAge) => {
  return personAge >= 18
}

if (getCanVote(17)) {
  console.log("We can vote!")
}

Reversing Conditionals with !

Sometimes you want to flip a boolean. That's what the Not operator does. By putting a ! next to a value, it will invert it. So !true is false, and !false is true. 

let age = 32
let canVote = age >= 18
let cantVote = !canVote
console.log("can't vote:", cantVote)

// or directly on a conditional
let alsoCantVote = !(age >= 18)
console.log("alsoCantVote", alsoCantVote)

Notice we used () in the second example. If we did !age >= 18 then that would mean we only reversed the boolean value of age. Sometimes that's what we want, but when you want the entire conditional, surround it with ().

Truthy and Falsy Values

Pop quiz: what do you think would happen if we did this: 

if ("hi there") {
  console.log('???')
}

We just put a non-boolean String in our conditional. Does this break? No, actually. That's because when you put a non-boolean value into the conditional check, JavaScript converts it into its truthy or falsy value. Yes, those are real terms in programming.

So, what's truthy and what's falsy? Here are all the falsy values:

Value Example
Literal Falsefalse
An Empty String""
The Number Zero0
The "Nothing" Valuesnull, undefined
Not a NumberNaN

That's it, everything else in JavaScript is a truthy value. All positive and negative numbers are truthy. All full strings are truthy, even strings made of spaces like " ". EVERYTHING not listed above is truthy. Play around with truthy and falsy values below.

else if and else

Now that we fully understand conditionals, let's get back to building our If/Else Statement. If our question boils down to "Do this or that", then we'll just need an else statement. 

let chanceOfRain = 72;
if (chanceOfRain >= 70) {
  console.log('Bring an umbrella.')
} else {
  console.log('No umbrella!')
}

An else is used when all other cases should result in that code block running. That's why there's no conditional next to the else. What you're saying is, "If the main conditional is false, run the else block." But what if we want more options? That's where the else if conditions come in

An else if condition allows you to specify for more conditionals. 

let pet = 'cat'

if (pet === 'cat') {
  console.log("My favorite pet!")
} else if (pet === "dog") {
  console.log("My 2nd favorite pet!")
} else if (pet === "snake") {
  console.log("Hard pass.")
} else if (pet === "turtle") {
  console.log("Love 'em!")
} else {
  console.log("Your pet is meh.")
}

Remember, each If statement starts a new chain. And in each chain, only one code block can execute. So, it helps to be specific.

Reassigning variables

Look again at that above example. That's a lot of console.logs, right? What if we just saved a variable called msg (universal shorthand for "message"), and then logged that only once? 

let pet = "cat"
let msg = ""

if (pet === "cat") {
  msg = "My favorite pet!"
} else if (pet === "dog") {
  msg = "My 2nd favorite pet!"
} else if (pet === "snake") {
  msg = "Hard pass."
} else if (pet === "turtle") {
  msg = "Love 'em!"
} else {
  msg = "Your pet is meh."
}
console.log(msg);

That's a little cleaner, but it looks like we added lines! But we aren't done yet. If we use the old else value instead of using an empty string, then we can remove else section. 

let pet = "cat"
let msg = "Your pet is meh."

if (pet === 'cat') {
  msg = "My favorite pet!"
} else if (pet === "dog") {
  msg = "My 2nd favorite pet!"
} else if (pet === "snake") {
  msg = "Hard pass."
} else if (pet === "turtle") {
  msg = "Love 'em!"
}
console.log(msg);

Our code still does the same thing, but now it's easier to read. When you change the way code is written, but leave it's functionality alone, that's called refactoring.

Return vs else

There's a another case were we don't need an else statement and that's when we're returning in a function. As you know, the return keyword causes us to skip whatever was left in the function. So if you are returning conditionally, then you may not need an else. 

const petOpinion = (pet) => {
  if (pet === 'cat') {
    return "My favorite pet!"
  } else {
    return "Your pet is meh."
  }
}

// that's the same as:
const petOpinion = (pet) => {
  if (pet === 'cat') {
    return "My favorite pet!"
  }
  return "Your pet is meh."
}

Multiple conditionals

So we have one conditional per block, yes, but what about a second conditional? Don't worry, we have special logical operators to deal with combos.

The OR operator ||

If you want something to happen as long as at least one condition is true, use the OR operator. It's two "pipe" characters ||. On most keyboards you get that by holding shift and the "\" key, which is usually one above the enter key. Here's how you use it: 

let pet = "cat"
let msg = "Your pet is meh."

if (pet === 'cat' || pet === 'dog') {
  msg = "I love your pet!"
}
console.log(msg);

That doesn't seem terribly intuitive, I know that I wished it was like: 

if (pet === 'cat' || 'dog') {
  msg = "I love your pet!"
}
NO THAT'S WRONG

But what you have to remember is conditionals are each their own boolean, so the OR operator is saying either conditional, not either value.

The AND operator &&

If you want every single condition to be true, then you need the AND operator. It's just two ampersands. 

let pet = "dog"
let name = "fido"
let msg = "You are creative"
if (name === "fido" && pet === "dog") {
  msg = "Ya basic"
}
console.log(msg)

If your pet is a dog but it's name isn't "fido", then that if statement won't fire.
