Lesson 1: Variables

How To Save Information

In programming, we need ways to save and reuse pieces of information. Like a character's name or their age. One way we can store data is by using a variable. Here is an example of a variable called myMood that has a value of Happy:

As you can see we have the name of myMood and then we set it equal to "Happy" using the = symbol. Makes sense. But what's that let doing out in the front? In JavaScript, in order to declare, or "create", a variable you have to use a special keyword first.

let Allows Change

If we don't mind if the variable changes its value, say 100 to 101, then we use the let keyword. 

let myNumber = 100
myNumber = 101

Notice that when we change the variable's value, we do not use another let keyword. We only use the let keyword once.

const Means Constant

If we want to lock the value in and never change it, we would use the const keyword. 

const myNumber = 100
myNumber = 101
THIS WILL THROW AN ERROR IT WILL NOT WORK

So which to use let or const?

My personal recommendation is to take it easy on yourself and mostly stick to let when your starting out. const errors from reassigning aren't hard to spot, but when starting out "simple" problems can still get annoying. Once you're more comfortable with code, you can definitely start using const properly, but don't sweat it if you just want to use let exclusively for now.

Regardless of let or const, don't forget that you can't reference a variable before you've actually created it! 

myNumber = 100
let myNumber = 101
NO GOOD, YOU USED myNumber BEFORE YOU DEFINED IT

Data Types

So, up until now we've exclusively used numbers or text for our variables, but that's not the only things we can store. In Programming, there are many different kinds of data types that we can store, but here are some of the simple ones to start us off.

Numbers

In JavaScript one of the most common values you'll find are Number types. And unlike other programming languages, it doesn't care if a number is a whole number (also called an "Integer") or a decimal (sometimes referred to as a "Float"). 

12
14242.23

You can also do math with javascript, here are some of the basic operations 

+ = addition
- = subraction
/ = division
* = multiplication

There's also a real handy trick when doing simple math operations. When making a variable bigger, a variable can reference itself to calculate a new value. Take this example: 

let a = 5
a = a + 10

Our variable a is now 15. I know the "referencing itself" thing is weird, but you get used to it. Anyway, there's actually a shortcut to do that same exact thing. 

let a = 5
a += 10

Once again, a is now 15. And it's not just addition. The +=, -=, /=, and *= assignment operators are a great way to increment or decrement values.

Obviously, you can do a lot with math and numbers in programming, but for our purposes this is good enough for now.

Strings

In JavaScript a piece of text is called a String. A string is a weird name for text, but thing about it like a piece of text is a bunch of individual characters "strung" along one after the other. Strings are created by putting anything between either double (") or single quotes('). Either one is fine, just make sure they match! 

let string = "I am a string"
let otherString = 'So Am I'

The only tricky thing with strings at this stage is making sure if you use a single quote for a word like "don't", be sure to use double quotes for the whole string so javascript will know you don't want the string to end. 

let tricky = "Well, I'm going to use double quotes"

There Is Another...

Using quotes is great, but there's a problem. What if we wanted to put a piece of data into a string that changes? There's actually a another way to make strings with dynamic parts, but you'll learn about later, stay tuned!

Booleans

That is a wonky name, but this is just what we call true and false in programming. Also called a "bool" for short. In javascript, bools are lowercase. And don't put them in quotes, they aren't text, they're actual values. 

let myBoolean = true
myBoolean = false

Booleans are primarily used for something called Flow Control, which is how a program decides whether or not something should be done. Just like humans do, we take in information before making a decision. Like, if it is raining, we bring an umbrella, otherwise we do not. But more on that later!

Undefined and Null

Sometimes the value we want to save is ...no value. Just like with numbers, where it's useful to have a value for nothing 0, with variables you sometimes want to explicitly say a value is nothing. Not 0, but actually nothing. That's what undefined and null are. There is a difference between them, but don't worry about it for now. 

let nothing = undefined
let alsoNothing = null

At this stage in the game, we likely aren't going to intentionally use either of them, however it's good for you to know about undefined specifically becuase of errors. Sometimes when JavaScript expects something but gets nothing, the value will show up as undefined. This is your hint that something is going wrong in your pogram and you should investigate further.

Comments

These aren't really a data type exactly, but they are useful to know. a comment is a chunk of text that is not seen by the program. It's useful for writing notes to yourself or your team members. Comments are an excellent way to keep relevant information about the program next to the source code, just be careful not to overdo it. 

// I am a single line comment

/*
  I am a multi line comment,
  anything inside me won't
  be seen by the program!
*/

// you can also stack
// single line comments
// together like this
// (This is more common)
If you're wondering why the stacked single line comments are more common, it's a good question. The reason is becuase most code editing programs have a special command to comment out highlighted code, and going line by line allows for much more precision. Replit does this too, so you'll see why it's handy when working on the the challenge.

A Quick Peek Ahead

There are 2 slightly more complex data types called Objects and Arrays. An Array is simply a list of things. 

let people = ['Bob', 'Sara', 'Jake']

They're great, but we don't need them yet. The one we will kind of need is the Object. An Object is a bunch of key and value pairs. Where a key (usually called a property) points to a specific value.

Or to put it another way, use dot notation with a property name to get a value. 

let bob = {
  name: 'Bob',
  age: 32,
  isCool: true,
}
bob.name
// This would be 'Bob'
bob.age
// this would be 32
bob.isCool
// and of course this would be true

We will not be writing any objects ourselves just yet, however, we will be using some properties of objects that JavaScript gives us. All you really need to know is: to access a value on an object, put a "." and then the property name. Don't worry, you'll understand better when we need to use them for real later on.
