Lesson 4: Strings

Say Something Good

String manipulation is one of the cornerstones of programming. Which makes sense since so much of what we do on the internet is just some kind of text. In this lesson we'll learn how to deal with strings and do some very useful things. Like, how to make them dynamic!

Template Strings

So often you'll only want a piece of a string to change. You can use something called concatenation to literally add strings together, but that's old and boring. Instead we use Template Strings, which allow us to interpolate (meaning "insert") data directly. Template Strings can also be called Template Literals if you're pretentious.

When creating a template string instead of using " or ', you surround your text with backticks ``. You get those by hitting shift and the key above your tab key with your left pinky. 

let greeting = `Hello!`

That's technically a template string, it's simply very boring. The entire thing is non-dynamic code, we could've just used quotes. To spice it up, let's add a variable to our greeting. Perhaps a name? In order to insert a chunk of code into the template, you have to wrap it inside a ${ } operator. You'll know you did it right if the inside code is a different color. 

let name = "Jay"
let greeting = `Hello ${name}!`

You can interpolate multiple pieces into a single string, and I meant it when I said "chunks of code." You aren't limited to only variables. Another thing to note is how spaces work. Essentially, the string prints exactly as you write it, the only thing to disappear is the ${} operator. Try playing around with some templates below to get the hang of it. 

let name = "Jay"
let emphasis = "!"
let greet = `Hello, ${name}${emphasis}`

console.log(greet)

let price = `$${4 + 2} dollars`
console.log(price)

Some other cool things about templates is that you can use newlines in your code that won't show up in your final output. This can be handy when adding large chunks of text together. Also, since a template is enclosed by backticks, you can use regular quotes inside without issue. 

// Super handy trick
let spaces = `Even though I'm
on multiple lines
I'll print as one,
space-separated line.`

console.log(spaces)

// NOT ALLOWED
let broken = "This
would not
  work at all"

let quotes = `Also, "Quotes are fine!"
Fell free to 'mix and match too!'`
console.log(quotes)

The length Property

Sometimes, (like in the next lesson) you'll want to know exactly how long a string is. JavaScript has you covered here, as every string has a built in property length. To use it, all you need to do is tack .length at the end of any string. Just make sure it's after the quote! 

console.log("hi".length) // 2
console.log('hello'.length) // 5
console.log(`hey`.length) // 3

let str = "hey hey"
console.log(str.length) // 7

let name = 'bo'
console.log(`hi ${name}`.length) // 5

To run this CodePen, make sure you're connected to the internet 4.3 Length Property

Take special note of that last template example. The length of the string is the final string, not the pre-computed version with the code inside it. Also remember that it's counting the total number of characters, so spaces and punctuation count towards the total.

String Indexes

So we talked about how we call text strings because it's like you're stringing a bunch of individual characters along. So it shouldn't surprise you to learn that JavaScript keeps track of the order of those characters. And we can access characters by referencing their index.

Each index is a number, and it only ever refers to a single character. Also, instead of starting at 1, indexes start at 0. So the 0th index is always the first character of the string.

Unlike properties, which we access via dot notation, indexes are accessed by bracket notation (sometimes also called Subscript Notation if you're coming from a math world). Which looks like this: 

let str = "hello"
console.log(str[0]) // h
console.log(str[2]) // l
console.log(str[4]) // o

// the index value can use any expression
// as long as it evaluates to a number
console.log(str[2]) // l
console.log(str[1+1]) // l

// you can access on a plain string too
console.log("wow"[0]) // w

Getting the last character

Here's a fun trick. Since we know that the last index will always be 1 less than the length (because length starts counting from 1, and indexes start from 0), there's a clever bit we can do to get the last character of the string. 

const getLastChar = (str) => {
  return str[str.length - 1]
}

console.log(getLastChar("Hey!")) // !
console.log(getLastChar("Oh?")) // ?

Immutable Strings

I don't want to get too in depth here, but strings themselves are immutable, meaning they can't be mutated or changed in any way. You can reassign a variable from one string to another, but you can't alter an existing string. For now, that means you can only use the indexes to read values, not set them. 

let str = "hello"
str[0] = "Z"

console.log(str)
// still says hello, not Zello

// but something like this is fine
let newStr = "hi"
newStr = `${newStr[0]}${newStr}${newStr[1]}`
console.log(newStr) // hhii

Common String Methods

Strings have built in properties like length, but they also have built in functions too! You access them with dot notation just like you do with length. Just don't forget the parentheses!

.toUpperCase() and .toLowerCase()

These two methods are for converting entire strings to upper or lower case. 

let greet = "hello"
let shoutedGreet = greet.toUpperCase()
console.log(shoutedGreet)
// HELLO

let loud = "AHHHH"
let sigh = loud.toLowerCase()
console.log(sigh)
// ahhhh

// remember, the originals are not changed
console.log(greet) // hello
console.log(loud) // AHHHH

Some common use cases are, only capitalizing the first letter, and lowercasing 2 strings in order to check equality. Because remember, === is case sensitive, so "Hello" !== "hello"

.slice()

Sometimes you only want a part of a string, that's what slice does. It returns a copy of a string. You can specify both the starting index, which is inclusive, and optionally the ending index, which is exclusive. That means the value at the first index and everything up to but not including the end index get's copied. If no end index is given, slice will just automatically copy to the end of the string. and if no indexes are given, the entire string is copied. 

let str = "Here I Am"
let copy = str.slice()
let here = str.slice(0,4)
let i = str.slice(5,6)
let am = str.slice(7)

console.log(copy) // Here I Am
console.log(here) // Here
console.log(i) // I
console.log(am) // Am

Now do you see why the .slice() method would work so well with the capitalize() function?

.includes()

This method is returns a boolean, true if the argument string is in the owner string, and false otherwise. 

let str = "Look at me!"
console.log(str.includes("Look")) // true
console.log(str.includes("at me")) // true
console.log(str.includes("ook ")) // true
console.log(str.includes("us")) // false

Just remember that this method is case sensitive, so when searching strings, it's a good idea to lowercase everything like before. 

const doesContain = (mainStr, searchStr) => {
  return mainStr
    .toLowerCase()
    .includes(searchStr.toLowerCase())
}

let wonky = "DoEs ThIs WoRk?"
let work = "work"
let isTrue = doesContain(wonky, work)
console.log(isTrue) // true

Chaining

Now you may look at that last example and see this: 

return mainStr
  .toLowerCase()
  .includes(searchStr.toLowerCase())

And rightly wonder "What the hell is that??" It's a valid question, but it's a really cool answer. It's called chaining, it's like a combo from a video game. Essentially, all these methods can be called on strings. So if a method returns a string, we can immediately call another method on it. Here's the same logic, but spread out. And as for the new lines with each method, that's totally fine as long as you start each line with the dot. 

// With chaining
const doesContain = (mainStr, searchStr) => {
  return mainStr
    .toLowerCase()
    .includes(searchStr.toLowerCase())
}

// no chaining
const doesContain = (mainStr, searchStr) => {
  let lowMainStr = mainStr.toLowerCase()
  let lowSearchStr = searchStr.toLowerCase()

  return lowMainStr.includes(lowSearchStr)
}

So close right? In all honesty, it comes down to a style preference. For a simple 2-3 chain, I'd probably recommend just making variables, however in the real world sometimes you can have quick 4-5 method chains full of inline functions, and it's just easier to chain things together.

The only wrong-ish choice is trying to one line something like that. That's just hard to read. 

const doesContain = (mainStr, searchStr) => mainStr.toLowerCase().includes(searchStr.toLowerCase())
